# Submission Mapping (เทียบกับตัวอย่างโปรเจค)

อ้างอิงตัวอย่าง: `https://github.com/annnoftx/2566-CPE311-Vinsmoke`

## ตารางเทียบโครงส่งงาน

| ในตัวอย่าง | ในโปรเจคเรา | สถานะ |
|---|---|---|
| README ที่รวมภาพรวม + วิธีใช้ + flow + gantt | `README.md` + `docs/submission_pack/README.md` | พร้อม |
| Presentation file | `docs/project_presentation_abstract.md` | พร้อม (ควร export เป็น PDF เพิ่ม) |
| Gantt chart | `docs/gantt_chart_th.md` | พร้อม |
| Flowchart | `docs/flowcharts.md`, `docs/full_system_flow_detailed.svg`, `docs/flowcharts_svg/README.md` | พร้อม |
| Project description + test cases | `docs/project_description_requirements_details_testcases_th.md` | พร้อม |
| Architectural design (actual update draft) | `docs/architectural_design_actual_update_draft_th.md` | พร้อม |
| Block diagram / circuit image แยกไฟล์ | `docs/block_diagram.md` + อ้างอิงใน abstract/architecture | พร้อม (ถ้าต้องส่งเป็นรูป ให้ export PNG/SVG เพิ่ม) |
| Test result evidence | อยู่ในเอกสาร description/abstract | ควรแยกหลักฐานเป็นไฟล์เฉพาะ |

## ช่องว่างที่ควรปิดก่อนส่งจริง

1. export `docs/block_diagram.md` เป็น PNG/SVG หากแบบฟอร์มต้องการไฟล์รูปโดยตรง
2. export ไฟล์สรุปนำเสนอเป็น PDF (`project_presentation_abstract.pdf`)
3. เก็บหลักฐานผลทดสอบแบบ screenshot/log ลงไฟล์เดียว
